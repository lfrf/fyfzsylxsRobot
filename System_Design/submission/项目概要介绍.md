# 虚拟情感交流机器人系统（A22）项目概要介绍

## 一、项目背景
随着学习与工作节奏加快，用户在日常生活中常出现压力大、情绪低落、缺少倾诉对象等问题。传统文本问答系统虽能提供信息，但缺少“可感知、可表达、可陪伴”的交互体验。本项目面向“服务外包创新创业大赛 A22 赛题”，构建一套可落地的虚拟情感交流机器人系统，实现从多模态输入理解到数字人语音视频回复的闭环能力。

## 二、项目目标
本项目目标是实现一个可运行、可演示、可部署的情感陪伴系统，具备以下核心能力：

1. 支持文本/语音输入，必要时可叠加视觉信息输入。
2. 通过大语言模型生成具有安抚和支持导向的回复内容。
3. 将回复文本经 TTS 生成语音，再驱动数字人生成对应 MP4 视频。
4. 以前后端分离方式提供网页端交互，支持远端 AI 服务接入和实时展示。
5. 支持基础工程化交付：脚本化启动、健康检查、可复现部署。

## 三、系统核心功能
### 1. 情感对话
- 支持用户文本/语音提问。
- 系统结合对话上下文与策略模板生成回应。
- 回复风格可体现“倾听、支持、鼓励”等情绪导向。

### 2. 数字人视频回复
- 使用 SoulX-FlashHead 数字人模型进行面部与口型驱动。
- 后端输出完整回复视频与分段流式视频清单，前端可边到边播。

### 3. 语音合成与播放
- 使用 CosyVoice-300M-Instruct 进行中文语音合成。
- 支持回复音频与视频联动输出，构建更自然的陪伴体验。

### 4. 多服务协同
- 远端采用多服务架构（LLM / Speech / Vision / Avatar / Orchestrator）。
- 本地前端通过代理/隧道与远端统一编排服务通信。

## 四、技术架构概述
系统采用“本地交互 + 远端推理”的分层架构：

- 本地层：Web 前端、边缘代理服务、浏览器交互。
- 远端层：五个核心 AI 服务及编排服务。
- 模型层：LLM、ASR、TTS、视觉理解、数字人驱动模型。

通过统一接口协议，编排服务将用户输入转化为结构化任务，调用各子服务产出最终多模态结果（文本+音频+视频）。

## 五、项目创新点
1. 从“纯文本问答”升级为“数字人情感陪伴”交互形态。
2. 实现“文本生成—语音合成—视频驱动”的完整工程闭环。
3. 支持分段视频输出，降低首帧等待时间，提升交互感知速度。
4. 可扩展多数字人配置，支持画像化与场景化演示。

## 六、应用价值
本项目可应用于：

- 校园心理支持与减压陪伴场景；
- 企业员工情绪关怀与热线辅助场景；
- 智能客服中“安抚型”交互前台；
- 医疗/康养场景中的轻量陪伴助手。

## 七、当前完成情况
截至当前版本，系统已完成：

1. 远端五服务启动与联调；
2. 本地前端与远端服务连通；
3. 文本到数字人视频回复链路可运行；
4. 基础文档、脚本与演示流程可复现。

## 八、后续优化方向
1. 持续提升语音稳定性与一致性控制；
2. 完善情绪识别与检索增强（RAG）策略；
3. 优化响应延迟和视频流式播放体验；
4. 强化评测指标体系（响应时延、可用性、用户满意度）。

## 九、结论
本项目围绕 A22 赛题完成了从算法能力到工程系统的落地，实现了“可对话、可表达、可演示、可部署”的虚拟情感交流机器人原型，具备实际推广与二次开发价值。
